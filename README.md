# 태그
## 다양한 문서 태그
* 'h1~h6' : 블록, 제목(대중소)
* 'p' : 블록, 내용(1줄~여러줄)
* 'hr' : 블록, 구분선(디자인선 아님 주의)
* 'br' : 인라인, 강제줄바꿈
* 'em' : 인라인, 특정 문맥 강조
* 'strong' : 인라인, 경고&할인 강조
* `code` : 인라인,프로그래밍 코드 구분용
* `sup`, `sub` : 인라인, 위첨자&아래첨자
* `q` `blockquote` : 인라인(q) 짧은인용, 블록(blpckquote) 긴인용문
* `s`, `del` : 인라인, s-유효하지 않는 내용, del-교체되거나 삭제된 내용
# 특수문자태그
* `&` : 특수문자태그의 시작기호
* `;` : 특수문자태그의 종료기호
* `&lt`,`&gt` : 왼쪽꺽쇠(<), 오른쪽꺽쇠(>)
* `&copy` : © copyright 저작권 기호

# 다양한 문서 태그-2
## block tag
* `div` : 2개 이상 자식을 묶는 그룹, class 또는 id 필수
* `ul-li` : ul(비순차그룹), li(목록) 엄격한 부모 자식-관계
* `ol-li` : ol(순차그룹), li(목록) 엄격한 부모 자식-관계
* `dl-dt-dd` : dl(정의형그룹), dt(정의형제목), dd(정의내용)
  * 검색키워드로 사용하지 않는 소제목과 내용 `h-p`를
  `dt-dd` 로 대체가능
  * `a` : 블록의 부모로 사용 시 링크 (div 등 그룹 대체 가능)
## inline tag
* `span` : 인라인 그룹 또는, 디자인이 다른 요소를 묶기 
* `a` :  인라인의 형제로 사용시 링크
### 비슷한 의미의 대체 태그들
* `h1~h6` vs `dt`
  * 중요도가 높고 검색키워드로서 사용되야 한다면 h태그
  * 제목-내용이 바로 연결되고 중요도가 낮으면 키워드로서
  사용가치가 낮을 떄 dt 태그
* `p` vs `li` vs `dd` vs `span`
  * 제목 하단에 단순 단락으로 구성된 내용 표시 p태그
  * 2개 이상 연속된 목록 구성 시 li태그
  * 중요도와 키워드가 낮은 제목 dt옆 내용을 감쌀떈 dd
  * 인라인으로 추가로 감싸야하는 내용이 있을땐 span
* `div` vs `ul` vs `ol` vs `a` vs `dl` vs `span`
   * di-2개 이상의 블록or인라인 자식으로 있을 떄 그룹
   * ul-순서가 없는 목록이 2개 이상있을때 그룹
   * ol-순서가 있는 목록이 2개 이상있을때 그룹
   * a-링크로 선택되는 태그가 2개 이상있을떄 그룹
   * dl-중요도가 낮은 제목-내용(dt-dd)가 있을 때 그룹
   * span-2개 이상의 인라인을 묶어야 할때 그룹
## 의미적인 그룹 **시멘틱태그**
### `자체적으로 의미가 없어서 클래스, id가 필요한 그룹`
* div, ul, ol, span, dl 등
### 태그명부터 의미로 구성되어 중복되지 않는한 class, id 가 필요없는 그룹
* header, nav, main, dside, footer, address 등
* header : 로고, 주요 메인메뉴 감싸는 그룹, 주로 최상단 위치
* nav : header안에서 메인메뉴를 감싸는 그룹, 서브(보조)메뉴엔 사용안함
* main : header와 footer 사이 내용 전체 묶는 그룹 (페이지당1번)
* footer : main이 끝나면 화면 최하단 영역 묶는 그룹(주소 저작권 등)
* aside : main안에서 좌,우측 사이드 묶는 그룹
* address : footer안에서 회사 주소 및 연락처 묶는 그룹
